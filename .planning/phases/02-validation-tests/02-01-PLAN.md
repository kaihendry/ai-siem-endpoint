<objective>
Add required-field validation and body size limiting to the POST / handler in main.go.

Currently handlePost validates run_id but does not check timestamp or host, and does not enforce a body size cap. This plan adds those checks so the endpoint returns HTTP 400 for missing required fields and HTTP 413 for payloads exceeding 1 MB.
</objective>

<must_haves>
  <truths>
    - POST / with missing run_id returns HTTP 400 with JSON error
    - POST / with zero-value timestamp (time.Time{}) returns HTTP 400 with JSON error
    - POST / with empty host field returns HTTP 400 with JSON error
    - POST / with body > 1 MB returns HTTP 413 before JSON decoding
    - Valid payloads continue to return HTTP 201 as before
  </truths>
</must_haves>

<tasks>

<task id="T01">
  <name>Add body size limit (1 MB) to handlePost</name>
  <read_first>
    - main.go (current handlePost implementation at line 112)
  </read_first>
  <action>
    In handlePost, before calling json.NewDecoder, wrap r.Body with http.MaxBytesReader:
    r.Body = http.MaxBytesReader(w, r.Body, 1<<20)
    Then in the json.Decode error path, check if the error is a *http.MaxBytesError (or the string "http: request body too large") and return HTTP 413 instead of 400.
    Use errors.As to detect *http.MaxBytesError specifically. Import "errors" if not present.
  </action>
  <acceptance_criteria>
    - main.go contains http.MaxBytesReader(w, r.Body, 1<<20) in handlePost
    - main.go contains a check for *http.MaxBytesError that writes StatusRequestEntityTooLarge (413)
    - "errors" is imported in the import block
  </acceptance_criteria>
</task>

<task id="T02">
  <name>Add timestamp and host validation to handlePost</name>
  <read_first>
    - main.go (handlePost at line 112, AuditRun struct at line 42)
  </read_first>
  <action>
    After the existing run.RunID == "" check (around line 119), add two more checks:
    1. if run.Timestamp.IsZero() → return HTTP 400 with error "invalid payload: timestamp is required"
    2. if run.Host == "" → return HTTP 400 with error "invalid payload: host is required"
    Keep the existing run_id check as-is. Order: run_id check, then timestamp check, then host check.
  </action>
  <acceptance_criteria>
    - main.go contains run.Timestamp.IsZero() check that writes StatusBadRequest with "timestamp is required"
    - main.go contains run.Host == "" check that writes StatusBadRequest with "host is required"
    - Both checks appear between the run.RunID check and the putEvent call
  </acceptance_criteria>
</task>

</tasks>
